**Задача:** синхронизировать `highlights.md` в корне репо с текущими highlight-ами в wiki/. Один запуск = один проход. Проект — `/Users/teorusak/Projects/course-wiki/`. Работаем прямо в `main`, без worktree.

**Контракт `highlights.md`:**

`highlights.md` содержит два типа записей:

1. **Auto-записи** — обёрнуты в маркеры:
   ```
   <!--auto:wiki/относительный/путь.md-->
   > цитата
   
   контекст
   <!--/auto-->
   ```
   Это записи, привязанные к конкретному `==...==` highlight-у в указанной wiki-статье. Routine ими управляет.

2. **Ручные записи** — всё, что вне auto-маркеров. Любой текст, любые цитаты, любые заголовки. Routine их **никогда** не трогает: не редактирует, не переставляет, не удаляет, не комментирует.

Заголовки `## [[wiki/...|Название]]` группируют записи по статьям-источникам и сами по себе не являются ни auto, ни ручными — это структурные элементы файла. Routine может их добавлять (когда появляется первая auto-запись из новой статьи) и удалять (когда под заголовком не остаётся ни одной записи — ни auto, ни ручной).

**Главный инвариант:** routine не трогает ничего вне `<!--auto:...-->`...`<!--/auto-->` блоков, кроме:
- структурных заголовков `## [[wiki/...|...]]` (добавление/удаление по правилам ниже);
- `---` отделителей вокруг этих заголовков;
- финального `\n` файла.

Если сомневаешься, относится текст к auto или к ручному — считай ручным и не трогай.

**Шаги:**

1. `git pull origin main`.

2. Прочитай `highlights.md` целиком.

3. Просканируй wiki/ на highlight-ы:
   ```
   grep -rn "==" wiki/ --include="*.md"
   ```
   Парсинг: highlight в Obsidian — это парный маркер `==...==`. Длинная цитата может занимать несколько подряд идущих строк, каждая обёрнута своей парой `==...==` (markdown-перенос строки не ломает Obsidian-выделение). Объедини такие соседние строки в одну цитату — это один логический highlight. Игнорируй `==` внутри code-блоков (тройные бэктики).
   
   На выходе — для каждой wiki-статьи список её текущих highlight-ов: `(src=относительный_путь, quote=полный_текст_цитаты)`.

4. Найди все auto-блоки в `highlights.md` (между `<!--auto:src-->` и `<!--/auto-->`). Для каждого:
   - Прочитай `src` из открывающего маркера.
   - Прочитай цитату внутри (строки, начинающиеся с `> `).
   - Поищи match среди highlight-ов из `src` (см. шаг 3). Match — по подстроке: первые ~30 значащих слов цитаты из auto-блока должны входить как непрерывная подпоследовательность хотя бы в один из highlight-ов в `src`. Нормализуй пробелы и переносы строк перед сравнением.

5. **Удаление auto-блоков** (highlight исчез из исходной статьи):
   - Если auto-блок не находит match → удали весь блок целиком, от `<!--auto:src-->` до `<!--/auto-->` включительно, плюс одну сопровождающую пустую строку после, если она есть.
   - НЕ удаляй auto-блок, если совпадение есть, даже частичное. Лучше оставить лишнее, чем выбросить нужное.

6. **Добавление новых auto-блоков** (highlight есть в wiki, но нет в `highlights.md`):
   - Для каждого highlight-а из шага 3, который не покрыт ни одним auto-блоком (ни существующим, ни ручной записью с похожей цитатой):
     - Прочитай ±15 строк контекста из исходного файла. Нужно понять: в каком разделе статьи (`##` / `###`) лежит цитата, и о чём конкретно эта мысль.
     - Сформулируй одно-два предложения контекста: что это за мысль, почему стоит держать в уме, в каком разделе статьи. Не пересказывай саму цитату, дополняй её. Регистр — спокойный, в духе текущего `highlights.md`. Не AI-голос.
     - Сформируй auto-блок:
       ```
       <!--auto:wiki/путь/к/файлу.md-->
       > цитата (можно в несколько строк через `> `)
       
       контекст
       <!--/auto-->
       ```
   - **Перед добавлением** проверь, нет ли в `highlights.md` ручной записи с похожей цитатой (тот же первый ~десяток слов). Если есть — пропусти, не добавляй дубль. Пользователь мог сознательно сделать цитату ручной.

7. **Размещение новых auto-блоков:**
   - Если в файле уже есть заголовок `## [[wiki/путь|Название]]` для этой статьи — добавь блок в этот раздел, в конце имеющихся блоков, перед следующим `---` (или перед следующим `## ` или EOF).
   - Если заголовка нет — создай раздел в конце файла. Формат:
     ```
     ---
     
     ## [[wiki/путь/к/файлу|Название]]
     
     <auto-блок>
     ```
     Название бери из первого `# ...` в исходном wiki-файле или из подписи в `wiki/index_do_course.md`. Wiki-link оставляй с относительным путём без расширения `.md` (как в текущих заголовках).

8. **Чистка осиротевших структурных заголовков:**
   - После всех удалений и добавлений пройди по файлу и найди заголовки `## [[wiki/...|...]]`, под которыми не осталось ни одной записи (ни auto-блока, ни ручного контента) до следующего `## `, `---` или EOF.
   - Удали такой заголовок и один сопровождающий `---` (тот, что отделяет его сверху или снизу — выбери так, чтобы не сломать визуальную структуру). Пустые строки вокруг тоже подчисть.
   - Заголовок без auto-блоков, но с ручным контентом под ним — НЕ трогай.

9. **Что НЕ делать:**
   - НЕ редактировать существующие auto-блоки (даже если контекст в них кажется неточным — это могла быть ручная правка пользователя внутри auto-блока, и она имеет приоритет).
   - НЕ трогать ручные записи. Ничего вне `<!--auto:...-->` ... `<!--/auto-->` не править, кроме осиротевших структурных заголовков (см. шаг 8).
   - НЕ переставлять разделы, не сортировать.
   - НЕ запускать filing, ingest, lens-добавление — отдельные процедуры.
   - НЕ читать `course/`, `course/hypothesis/`, `raw/`. К этой задаче не относятся.
   - Раздел «Гипотезы курса» в `wiki/index_do_course.md` и блоки `## В линзе гипотез курса` в статьях читать можно, если highlight стоит там — но контекст пиши в регистре автора, без курс-связки. Указывай, в каком разделе статьи цитата (например: «раздел ‘В линзе гипотез курса → H1’»).

10. **Если изменений нет** (нет ни добавлений, ни удалений) — ничего не делай, не коммить, выйди со строкой «highlights: без изменений».

11. **Если изменения есть** — закоммить и запушь:
    ```
    git add highlights.md
    git commit -m "highlights: +N -M"
    git push origin main
    ```
    где N — добавлено, M — удалено. В теле коммита — короткий список (`+ статья — новые цитаты` / `- статья — удалённые цитаты`).

**Отчёт пользователю в чат (короткий):**
- Сколько добавлено, сколько удалено, из каких статей.
- Если без изменений — одна строка.
- Не пересказывай цитаты, не описывай шаги.
